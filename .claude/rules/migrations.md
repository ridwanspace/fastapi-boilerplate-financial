# Migration Rules

- Always use `--autogenerate` from SQLAlchemy models — never hand-write DDL for model changes
- Migration command: `make migrate-create` (wraps `alembic revision --autogenerate -m "<description>"`)
- Apply migrations: `make migrate` (wraps `alembic upgrade head`)
- **Every financial table must include these columns** at creation:
  ```python
  sa.Column("version", sa.Integer(), nullable=False, server_default="0"),
  sa.Column("created_by_id", sa.Uuid(), nullable=True),
  sa.Column("updated_by_id", sa.Uuid(), nullable=True),
  sa.Column("deleted_at", sa.DateTime(timezone=True), nullable=True),
  ```
- **Never drop a column** without confirming it is unused in all environments
- For renaming: always add new column → backfill → switch reads → remove old column (three migrations)
- Review autogenerated migration before applying — check that `CheckConstraint` and partial indexes are included
- Migration filenames use the auto-generated revision prefix — do not rename them
